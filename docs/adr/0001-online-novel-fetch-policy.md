# 0001 — オンライン小説の取得ポリシー

**Status**: accepted (2026-05-27)

## Context

GeekPlayer は v0.1 で小説家になろう / ノクターン系 / カクヨムを対象としたオンライン
小説リーダー機能を提供する。3 サイトの API 提供状況は対称ではない:

| サイト | 検索/一覧 | 本文取得 |
|---|---|---|
| 小説家になろう | 公式 API | 公式 API |
| ノクターン系 | 公式 API (`novel18api`) | 同左 |
| カクヨム | 公式 RSS のみ | **公式手段なし** |

カクヨム本文の取り扱いには (a) アプリ内 WebView で公式ビューアを表示する、(b) v0.1 ではカクヨムを対象外にする、(c) HTML をパースしてアプリ内表示する、の 3 選択肢があった。

## Decision

**(c) を採用する。カクヨム本文は HTML パースで取得し、アプリ内で表示する。**
同時に、責任あるスクレイピングのために以下の運用規範を全サイト共通で固定する。

### 取得方針

1. **能動キャッシュのみ**: ユーザーが `Library` に追加した `Work` の本文だけがローカル DB に保存される。受動的なクロール/ミラーリングは行わない。
2. **TTL なし**、ユーザー手動削除のみ。デフォルトのキャッシュ上限は無制限、設定で MB 単位で上限を指定可能。
3. **カクヨム専用のレート制限**: 1 リクエスト / 2 秒、並列度 1。
4. **User-Agent**: `GeekPlayer/<version> (+https://github.com/geekjapan/GeekPlayer; personal-use)`。
5. **`robots.txt` を尊重**。
6. **429 / 503 で指数バックオフ**（最大 5 分）。
7. **同意ダイアログ**: 初回起動時にサイト別のチェックボックスで同意を取得。同意なしの `Site` の機能は無効化（カクヨムを拒否してもなろう/ノクターンの機能は利用可能）。

### 注意書きの掲示場所（4 箇所）

1. `README.md` の「カクヨム機能の注意事項」セクション
2. アプリ初回起動時の同意ダイアログ
3. `Settings > オンライン小説` 画面の常時表示文言
4. `KakuyomuHtmlSource` クラスの docstring

## Considered Options

- **(a) WebView フォールバック**: 法務的に最も安全だが、アプリ内のフォント/縦書き/しおり粒度が統一できない。オフライン読書も不可。
- **(b) カクヨムを非対応**: スコープを縮めて確実性を取る案。3 サイト統合体験が崩れるため不採用。
- **(c) HTML パース（採用）**: 体験価値が高い反面、ToS 上のグレー領域・サイト改修への耐性が低い。注意書きと能動キャッシュ運用で責任を持って提供する。

## Consequences

- `KakuyomuHtmlSource` の保守コストはカクヨム側の HTML 構造変更に応じて発生する。スナップショットテスト + 統合テストで早期検知する。
- 将来カクヨムが公式 API を提供した場合、この ADR は新 ADR で superseded する。
- 将来カクヨムが ToS で明示的に自動収集を禁止した場合、本機能は速やかに無効化し、(a) もしくは (b) に切り替える。
